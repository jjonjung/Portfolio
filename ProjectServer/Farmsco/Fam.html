1. 핵심 요약

인터페이스 실시간 데이터 처리 시스템 안정화

상태 기반 처리 구조와 재처리 가능한 운영 체계를 설계하여,
SAP–MES 인터페이스를 안정적인 데이터 파이프라인으로 개선한 프로젝트입니다.

▶ 프로젝트 개요

  • 역할: SAP–MES 인터페이스 설계 및 실시간 데이터 처리 시스템 개발
  • 기술 스택: C#, MSSQL, Stored Procedure, 멀티스레딩
  • 처리 규모: 일 평균 1만 건 이상 데이터 처리
  • 핵심 키워드: 상태 기반 처리, Idempotent 설계, 배치 처리, 인덱스 최적화, 오류 재처리, 운영 모니터링

▶ 시스템 배경

표준 MES 기능 모듈을 기반으로 고객사 프로세스에 맞는 시스템 구성을 이해하고,
SAP와 MES 간 인터페이스가 연결되는 운영 환경에서 실시간 데이터 처리 안정화 작업을
수행했습니다.

▶ 핵심 요약

▶ 프로젝트 배경과 문제 정의

기존 SAP–MES 인터페이스는 불안정했고, 장애가 발생하면 운영자가 직접 DB를 수정해야 하는 상황이 반복되었습니다. 단순 적재 방식의 구조에서는 어느 단계에서 처리가 멈췄는지 추적하기 어려웠고, 대량 데이터를 한 번에 처리하면서 Lock 대기와 성능 저하도 발생하고 있었습니다.
이 프로젝트에서는 문제를 단순 성능 이슈가 아니라, 신설 구축이기 때문에 테이블 구조·트랜잭션 범위·오류 복구 방식 자체의 설계 문제로 보고 접근했습니다. 

1. 현상 파악

  • SAP에서 MES로 데이터를 전송하는 인터페이스가 하루 3~4회 멈춤
  • 처리 중단 시 어디서 멈췄는지 추적이 어려움
  • 오류 발생 시 운영자가 수동으로 확인 후 DB 수정 필요
  • 대량 데이터를 한 트랜잭션으로 처리하며 Lock 대기 발생 

2. 원인 분석

  • 테이블 구조가 단순 적재용이라 처리 상태를 추적할 수 없음
  • 수신부터 저장, 완료 처리까지 한 번에 처리되어 책임이 과도하게 집중됨
  • 대용량 데이터 처리 시 트랜잭션 범위가 커져 병목이 발생함
  • 오류가 나도 운영자가 직접 개입하지 않으면 복구가 어려운 구조였음

▶ 수행 접근 방식

flowchart LR
    A[현행 구조 분석] --> B[GAP 분석]
    B --> C[To-Be 설계]
    C --> D[프로토타입 검증]
    D --> E[사용자 적용]
    E --> F[안정화]
반복적으로 발생하던 인터페이스 중단 문제를 해결하기 위해,
현행 구조를 분석하고 GAP을 도출한 뒤 To-Be 구조를 설계했습니다.
이후 프로토타입 검증과 사용자 적용, 안정화 단계까지 순차적으로 진행하며
실제 운영 가능한 구조로 개선했습니다.

▶ 핵심 성과

단순히 성능만 개선한 것이 아니라, 장애가 발생하더라도 추적·복구·재처리 가능한 운영 구조를 확보했다는 점이 가장 큰 성과였습니다. 
개선 항목변경 전변경 후성과
Copy 중단 빈도하루 3~4회주 1회 이하약 80% 감소
장애 대응 시간약 30분약 5분대응 속도 대폭 향상
DB Lock 대기높은 대기 발생70% 감소배치 처리 구조로 전환
운영 안정성제한적 복구 방식상태 기반 복구 구조시스템 안정성 및 복구력 향상

▶ 구현 요약

  • SAP–MES 인터페이스 전용 상태 관리 테이블 설계
  • WAIT / PROCESSING / DONE / ERROR 기반 처리 프로시저 설계
  • 배치 단위 트랜잭션 처리 구조 적용
  • 빈번 조회 컬럼 기준 인덱스 재구성
  • C# 기반 수신/검증/저장/상태 갱신 로직 구현
  • 오류 로그 저장 및 재처리 기능 구현
  • 운영 현황 대시보드 및 오류 상세 조회 기능 개발 

▶ 프로젝트를 통해 얻은 기술적 인사이트

이 경험을 통해 실시간 데이터 처리 시스템에서 중요한 것은 단순히
“문제가 발생하지 않게 만드는 것”이 아니라, 문제가 발생해도 어디서 멈췄는지 추적하고 다시 복구할 수 있는 구조를 설계하는 것이라는 점을 배웠습니다.
또한 트랜잭션 범위 설계, 상태 기반 처리, 멱등성 확보, 로깅과 모니터링 체계가 운영 안정성을 좌우한다는 것을 실무에서 직접 체득했습니다. 

▶ 직무 역량

이 프로젝트는 제조 도메인 사례이지만, 기술적으로는 실시간 데이터 처리 구조 설계, DB 상태 관리, 배치 단위 트랜잭션 제어, 재처리 가능한 오류 대응 구조, 운영 모니터링 체계 구축 경험을 담고 있습니다. 이는 도메인이 달라도, 라이브 서비스형 서버에서 요구되는 정합성·복구 가능성·운영 안정성과 직접 연결되는 역량이라고 생각합니다.
